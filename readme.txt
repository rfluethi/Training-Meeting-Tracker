WordPress plugin that displays the DACH training team's meetings on a WordPress page, sourced from GitHub issues.

== Description ==

This plugin fetches a centrally maintained JSON file with the DACH training team's meeting overview and renders it on the WordPress site via shortcode.

The JSON is generated by a GitHub Action in the team repository from all issues labelled "sitzung" and lives on a dedicated data branch. The plugin loads this file over HTTP, validates it against a schema, and caches it as a transient.

Features:

* Shortcode `[training_meeting_tracker]` with attributes `show_upcoming`, `show_in_progress`, `show_past`, and `years`.
* "Upcoming meetings" list: every issue without the "Erledigt" label whose date lies in the future.
* "Meetings in progress" list: every issue without the "Erledigt" label whose date already passed (waiting for minutes).
* List of past meeting minutes, grouped by year (issues labelled "Erledigt").
* Date format on the frontend: DD.MM.YYYY.
* Settings page under Settings, Training Meeting Tracker, with JSON URL, cache duration, and a clear cache button.
* Falls back to the most recent successfully loaded data if the source is temporarily unreachable.
* Cleans up its own options and transients on uninstall.

Source code and contributions: https://github.com/rfluethi/Training-Meeting-Tracker

This plugin is the successor of "Learn DACH Sitzungen" (repository `learn-wp-dach-sitzungen`). The predecessor is archived; all further development happens here.

== Installation ==

1. Upload the plugin ZIP under Plugins, Add New, Upload Plugin, or copy the `training-meeting-tracker` folder to `wp-content/plugins/`.
2. Activate the plugin.
3. Under Settings, Training Meeting Tracker, verify the JSON URL and adjust if needed.
4. Place the shortcode `[training_meeting_tracker]` on a page or post.

== Frequently Asked Questions ==

= Where does the data come from? =

From a JSON file generated by a GitHub Action in the team repository, built from all issues with the label "sitzung". The default URL points to the data branch of the repository `rfluethi/learn-wp-dach-team`.

= What happens when GitHub is unreachable? =

The plugin keeps the most recent successful data set in an option and renders it with a notice indicating that the data is not fresh.

= How do I show only the current year? =

`[training_meeting_tracker years="1"]`

= How do I hide one of the sections? =

`[training_meeting_tracker show_upcoming="false"]`, `show_in_progress="false"` or `show_past="false"`. The legacy `show_next` attribute is still accepted as an alias for `show_upcoming`.

= How long is the data cached? =

By default 12 hours. Configurable in the plugin settings between 1 and 168 hours. A "Refresh now" button is available on the settings page for an immediate manual refresh.
